# Contributing to OpenXR Quest Tutorial

Thank you for your interest in contributing to this project! We welcome contributions from the community to help improve the documentation and expand the collection of samples and tutorials.

## How to Contribute

We encourage contributions in the following areas:

-   **Improving Documentation:** Fixing typos, clarifying instructions, improving language, and adding missing details to any of the `.md` files.
-   **Adding New Samples:** Creating new, self-contained samples that demonstrate a specific OpenXR or Meta XR feature.
-   **Enhancing Existing Samples:** Adding features or fixing bugs in the current samples.
-   **Reporting Issues:** If you find a bug or a documentation error, please open an issue on our GitHub repository.

## Contribution Workflow

1.  **Fork the Repository:** Start by forking the main repository to your own GitHub account.
2.  **Create a Branch:** Create a new branch for your changes. Use a descriptive name, such as `fix/readme-typo` or `feat/add-passthrough-sample`.
    ```bash
    git checkout -b <your-branch-name>
    ```
3.  **Make Your Changes:** Make your desired changes to the code or documentation. Please ensure your changes adhere to the project's existing style and conventions.
4.  **Commit Your Changes:** Commit your changes with a clear and concise commit message.
    ```bash
    git commit -m "feat: Add new sample for Scene API"
    ```
5.  **Push to Your Fork:** Push the changes to your forked repository.
    ```bash
    git push origin <your-branch-name>
    ```
6.  **Create a Pull Request:** Open a pull request from your branch to the `main` branch of the original repository. Provide a clear description of the changes you've made and why they are valuable.

## Code and Documentation Style

-   **Code:** Please try to match the coding style, formatting, and architectural patterns found in the existing samples.
-   **Documentation:** Ensure that any new samples or features are accompanied by clear and concise documentation in a `README.md` file within the sample's directory. Follow the format of existing documentation.

We appreciate your contributions and look forward to your pull requests!
